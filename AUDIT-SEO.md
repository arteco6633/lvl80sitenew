## 1. Метаданные и теги в `<head>`

### Сделано хорошо
| Элемент | Статус |
|--------|--------|
| `lang="ru"` | ✅ Корректно задан на `<html>`. |
| `<title>` | ✅ «LVL 80 — разработка сайтов и digital» — осмысленный, с брендом и тематикой. |
| `<meta name="description">` | ✅ Есть, ~120 символов, ключевые слова (разработка сайтов, SEO, UX/UI, веб-разработка). |
| `<meta name="viewport">` | ✅ Настроен, адаптив учитывается при индексации. |
| `<meta charset="UTF-8">` | ✅ |
| Open Graph: `og:title`, `og:description`, `og:type` | ✅ Задают заголовок и описание при расшаривании. |

### Рекомендации
| Проблема | Решение |
|----------|---------|
| **Нет `og:image`** | Добавить `<meta property="og:image" content="https://ваш-домен.ru/og-image.jpg">` (рекомендуемый размер 1200×630). Без него соцсети не покажут превью. |
| **Нет `og:url`** | Добавить `<meta property="og:url" content="https://ваш-домен.ru/">` для однозначной привязки к каноническому URL. |
| **Нет Twitter Card** | Добавить `twitter:card`, `twitter:title`, `twitter:description`, при наличии — `twitter:image`. |
| **Нет канонического URL** | Добавить `<link rel="canonical" href="https://ваш-домен.ru/">` во избежание дублей при разных вариантах URL (с/без www, с/без слеша). |

## 2. Структура страницы и заголовки

### Иерархия заголовков
| Уровень | Где используется | Оценка |
|---------|-------------------|--------|
| **h1** | Один на странице: Hero — «Мы делаем сайты, которые [смена фраз]» | ✅ Один h1 — правильно. |
| **h2** | Проекты («Кейсы, за которые не стыдно»), CTA («Увеличиваем эффективность сайта»), ниши («Решение для роста бизнеса»), контакты («Связаться с нами») | ✅ Логичные блоки. |
| **h3** | Карточки услуг, тарифов, ниш, подзаголовки в футере и модалках | ✅ Используются по смыслу. |

### Замечания по разметке
| Секция | Проблема | Рекомендация |
|--------|----------|--------------|
| **Услуги** | Заголовок «От стратегии до запуска» в `<div class="strategy-section__ghost-title">`, не в заголовке | Для семантики и SEO лучше обернуть в `<h2>` (можно визуально оставить текущий стиль). |
| **Тарифы** | «Пакеты тарифов» в `<p class="tariffs-header__ghost">` | Имеет смысл сделать заголовком секции: `<h2 class="tariffs-header__ghost">`. |
| **FAQ** | Заголовок секции — «FAQ» в `<p class="faq-ghost">` | Для ясности структуры лучше `<h2 class="faq-ghost">`. |

**Итог:** Иерархия в целом корректна; небольшая доработка — вынести визуальные «заголовки» секций в теги h2.

## 4. Изображения

### Сделано хорошо
- У картинок проектов есть осмысленный `alt` (название проекта или `imageAlt`).
- Декоративные элементы: `alt=""` или `aria-hidden`, что корректно.
- Используются `loading="lazy"` и `decoding="async"` для части изображений.

### Рекомендации
| Элемент | Рекомендация |
|---------|---------------|
| Иконки ниш (`/niches-icon.svg`) | `alt=""` у декоративных иконок — ок; если иконка несёт смысл, задать короткий `alt`. |
| CTA-ракета | Сейчас `alt="Rocket"` — можно заменить на что-то вроде «Иллюстрация: запуск и рост» для контекста. |
| **og:image** | Подготовить отдельное изображение для соцсетей (логотип/баннер 1200×630). |

## 6. Семантика и доступность (влияние на SEO)

### Плюсы
- Используются теги: `<header>`, `<main>`, `<nav>`, `<footer>`, `<section>`, `<article>`.
- У модалок: `role="dialog"`, `aria-modal="true"`, `aria-labelledby`.
- У карточек проектов: `role="button"`, `aria-label` с названием проекта.
- У раскрывающихся блоков FAQ: `aria-expanded`, обработка клавиатуры.

Семантика и a11y помогают поисковым системам лучше понимать структуру страницы.

## 7. Технические аспекты для SEO

### SPA и индексация
- Контент подгружается через React (клиентский рендер). Google хорошо индексирует такие страницы при корректном рендере.
- Все важные мета-теги (title, description) находятся в статическом `index.html`, поэтому сниппет в поиске формируется даже до выполнения JS.
- **Рекомендация:** после публикации проверить страницу в «Просмотр как Google» (Search Console) или инструментами типа «Fetch as Google», чтобы убедиться, что контент и заголовки видны.

### Чего не хватает
| Элемент | Назначение | Рекомендация |
|---------|------------|--------------|
| **robots.txt** | Управление обходом краулеров | Положить в `public/robots.txt`: разрешить индексацию и при наличии — указать путь к sitemap. |
| **sitemap.xml** | Помогает поисковикам обнаружить страницы | Для одностраничника — одна запись с основным URL; при появлении новых страниц — дополнять. |
| **Canonical** | См. п. 1 | Обязательно после фиксации домена. |

## 8. Структурированные данные (Schema.org)

**Текущее состояние:** разметка типа JSON-LD не используется.

### Рекомендации по внедрению
Добавление схем повышает шансы на расширенные сниппеты и лучше передаёт смысл страницы.

| Тип | Где разместить | Что даёт |
|-----|----------------|----------|
| **Organization** | В `index.html` (или через React в `<head>`) | Указать название, логотип, URL, контакты. Может использоваться в Knowledge Panel. |
| **LocalBusiness** (при наличии офиса/города) | Аналогично | Локальные запросы, карточка в поиске. |
| **FAQPage** | Страница с блоком FAQ | Возможность вывода блока «Часто задаваемые вопросы» в выдаче. |
| **WebSite** + **SearchAction** (если будет поиск) | На главной | Расширенные возможности для сайта в поиске. |

Минимально полезный набор для лендинга: **Organization** + **FAQPage**.
